Spring AOP 教程

1. AOP 到底是什么？

在软件开发里，有很多"公共流程"是几乎所有业务方法都要做的，比如记日志、记耗时、查权限、管事务。如果每个业务方法都自己写一遍这些代码，会又臭又长还容易出错。AOP 的思路就是：把这些公共逻辑抽出来集中管理，业务方法只管自己的核心逻辑，公共逻辑由 AOP 在"合适的时机"自动切入。

1.1 没有 AOP 之前，代码长什么样

光说概念太虚，直接看代码。假设你有一个用户服务类，里面有几个方法，老板要求每个方法都要打印日志、统计耗时、校验权限。如果没有 AOP，你可能会这样写：

1.3 一句话定义 AOP

AOP（面向切面编程）是一种编程思想，它通过将横切关注点（日志、事务、权限等公共逻辑）从业务代码中抽离出来，集中放到切面中管理，再由框架在方法执行的特定时机自动织入，从而实现业务代码与公共逻辑的彻底解耦。

2. 核心概念

AOP 有一堆术语：切面、切入点、通知、连接点、织入、目标对象、代理……新手第一次看直接劝退。其实这些词对应到生活里都很好理解，我们用"快递员送快递"这个场景把它们串起来。

2.1 七大核心概念

术语 英文 白话解释 快递员类比
切面 Aspect 装着公共逻辑的类，比如日志切面 快递公司（制定送件规则的地方）
通知 Advice 切面在某个连接点要执行的操作，比如"打印日志" 快递员实际做的动作：签收、打电话
切入点 Pointcut 定义"哪些方法需要被切"，用表达式描述 今天要送的快递清单（地址匹配规则）
连接点 JoinPoint 连接点是方法调用或者方法执行时的某个特定时刻（如方法调用、异常抛出等） 每一个潜在的送件地址
织入 Weaving 把切面"贴"到目标方法上的过程 快递员按清单去对应地址执行动作
目标对象 Target 被切的那个原始业务对象 收件人所在的地址
代理 Proxy 织入后生成的"增强版"对象，外部调用的就是它 实际跑腿的快递员（代表公司）

看完这张表，再用一句话串起来：你写了一个切面（Aspect），里面定义了通知（Advice，具体干什么）和切入点（Pointcut，对谁干），Spring 在程序运行时找到所有匹配切入点的连接点（JoinPoint，方法），把通知织入（Weaving）到目标对象（Target）上，生成一个代理对象（Proxy），外部调用代理对象时就会自动执行通知逻辑。

2.2 通知的五种类型

通知（Advice）是切面里最核心的部分，它决定了"在目标方法的什么时机执行公共逻辑"。Spring AOP 提供了五种通知类型：

通知类型 注解 执行时机 典型用途
前置通知 @Before 目标方法执行之前 打印入参、权限校验、初始化资源
后置通知 @After 目标方法执行之后（无论成功失败） 清理资源、最终日志
返回通知 @AfterReturning 目标方法正常返回之后 处理返回值、记录成功日志
异常通知 @AfterThrowing 目标方法抛出异常之后 记录异常、告警通知
环绕通知 @Around 包裹目标方法前后（最强大） 性能监控、缓存、事务、限流

@After 和 @AfterReturning 不是一回事。@After 是"无论如何都执行"，类似 finally；@AfterReturning 是"只有正常返回才执行"，如果方法抛异常了它就不执行。

2.3 切入点表达式

切入点用一种叫"切入点表达式"的语法来描述要切哪些方法。最常用的是 execution 表达式，格式如下：

部分 说明 是否必填
修饰符 public / private / protected 可选
返回值类型 void / String / * 等 必填
包名 com.example.service / com.example… 必填
类名 UserService / * / *Service 必填
方法名 getUser / * / get* 必填
参数类型 () / (String) / (…) 必填
异常 throws Exception 可选

* 代表任意；… 在包路径里表示"当前包及子包"，在参数列表里表示"任意个数任意类型的参数"。

3. 五种通知详解：每种怎么写、什么时候用

3.1 @Before 前置通知

在目标方法执行之前执行。常用于打印入参、权限校验、初始化资源。

3.2 @After 后置通知（最终通知）

在目标方法执行之后执行，无论方法是正常返回还是抛异常，它都会执行。类似 try-finally 里的 finally。常用于清理资源。

3.3 @AfterReturning 返回通知

只有目标方法正常返回时才执行，抛异常则不执行。可以拿到方法的返回值，常用于处理返回结果、记录成功日志。

注意 returning 属性的值必须和方法参数名一致，否则 Spring 找不到要绑定的参数。这是新手常踩的小坑。

3.4 @AfterThrowing 异常通知

只有目标方法抛出异常时才执行，正常返回则不执行。可以拿到抛出的异常对象，常用于记录异常日志、发送告警。

3.5 @Around 环绕通知（最强大）

环绕通知包裹住整个目标方法的执行，你可以在方法前后做任何事，甚至决定要不要执行目标方法、修改参数、修改返回值。它是五种通知里功能最强的，也是实际开发中最常用的。

三个关键点新手必须记住：第一，参数必须是 ProceedingJoinPoint（不是 JoinPoint），它多了 proceed() 方法；第二，必须调用 joinPoint.proceed() 才会执行目标方法，不调用目标方法就不执行（这是实现缓存、限流的基础）；第三，必须 return 结果，否则调用方拿不到返回值。

5. 新手高频踩坑点：这些坑我都替你踩过了

AOP 用起来爽，但坑也不少。这一章盘点新手最容易踩的 7 个坑，每个都给出原因和解决方案。看完能少走很多弯路。

5.1 同类方法内部调用，AOP 不生效（最高频！）

为什么？因为 AOP 靠代理对象生效。外部调用 methodA 时，走的是代理对象，切面能拦到。但 methodA 内部调用 methodB 时，是 this.methodB()，this 是原始对象不是代理对象，所以 methodB 的切面拦不到。

解决方案有三种：

方案一：把 methodB 抽到另一个类里，通过注入调用（推荐，最干净）。

方案二：在类里注入自己，用 self.methodB() 调用。

方案三：从容器里拿到代理对象再调用：((UserService) AopContext.currentProxy()).methodB()，（需要在启动类或配置类上配置 @EnableAspectJAutoProxy(exposeProxy = true)）

5.2 private 方法无法被切

Spring AOP 基于代理，代理只能拦 public、protected 方法，private 方法拦不到。如果你给 private 方法加注解或写切入点匹配它，切面不会生效，而且不会报错，非常隐蔽。

解决方案：把 private 方法改成 public 或 protected。如果实在不想暴露，说明这个方法不该被切，重新考虑设计。

5.3 final 方法和 final 类无法被 CGLIB 代理

CGLIB 靠生成子类实现代理，final 类不能被继承、final 方法不能被重写，所以 final 方法/类的切面不生效。如果你发现某个类的切面死活不生效，检查一下是不是 final。

解决方案：去掉 final 关键字。如果是为了防止继承，用其他方式（如包级私有、文档约定）替代。

5.4 切面没生效—忘了加 @Component

切面类只加 @Aspect 是不够的，@Aspect 不会让 Spring 把它注册成 Bean。必须再加 @Component（或 @Configuration 等），切面才会被 Spring 管理，切入点才会生效。

5.5 多个切面执行顺序混乱

一个方法被多个切面切时，执行顺序默认是不确定的。如果你有日志切面和事务切面，可能日志在事务外、也可能在事务内，行为不一致。

解决方案：用 @Order 注解控制切面执行顺序，数字越小优先级越高（越先执行外层）。

5.6 @Around 忘记调用 proceed() 或忘记 return

写 @Around 时两个低级错误：一是忘了调用 joinPoint.proceed()，结果目标方法根本没执行；二是忘了 return result，调用方拿到的返回值是 null。这两个错误都不会报错，但行为完全不对，排查起来很折磨人。

解决方案：写 @Around 时养成模板习惯，先写 try-finally 框架，再填业务逻辑，确保 proceed() 一定被调用、return 一定有。
