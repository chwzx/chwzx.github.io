Zane Wang2026-06-262026-07-08

Java反射

反射（Reflection）是指 Java 程序在运行时，能够动态获取类的完整结构信息（如属性、方法、构造器），并能动态操作对象的属性和方法的能力。

在介绍如何使用反射之前，先定义一个 User 类。

1. 反射的常见操作

1.1 获取Class对象的三种方式

三种方式对比

方式 写法 确定时机 是否需要类已加载 典型场景
getClass() obj.getClass() 运行时 是（对象已存在） 已知对象，想获取其实际运行时类型
.class User.class 编译期 否（仅引用类） 类型已知，需要 Class 对象做参数
forName() Class.forName(…) 运行时 否（会触发加载） 类名字符串动态传入，如配置、反射框架

1.2 new 对象 vs 反射创建对象

1.21 new 创建对象（传统方式）

new 方式的特点：编译时就确定了要创建什么类，代码写死。如果要换一个类，必须改代码重新编译。这叫"静态创建"。

方法 作用 注意点
Class.getConstructor(…) 获取public构造器 只能获取 public

1.3 反射访问成员变量

读取/修改 static 字段

关键 API 对比

方法 作用
Class.getField(String name) 获取 public 字段（含继承的）
Class.getDeclaredField(String name) 获取本类任意权限字段（不含继承）
Field.setAccessible(true) 绕过访问检查，操作 private 字段
field.get(Object obj) 读取字段值
field.set(Object obj, Object value) 修改字段值
field.getType() 获取字段类型

1.4 反射获取成员方法

假设有 UserService 类：

关键 API 对比

方法 作用
Class.getMethod(String name, Class<?>… paramTypes) 获取 public 方法（含继承的）
Class.getDeclaredMethod(String name, Class<?>… paramTypes) 获取本类任意权限方法
Method.setAccessible(true) 绕过访问检查，调用 private 方法
method.invoke(Object obj, Object… args) 执行方法
method.getReturnType() 获取返回值类型

1.5 反射获取类信息

通过 Class 对象可以获取类的各种元信息：类名、包名、父类、接口、注解等。这在框架扫描类时特别有用。

实战一：手写对象拷贝工具类

业务场景：把源对象的字段值拷贝到目标对象的同名字段。比如把 UserDTO 的数据拷贝到 User 实体。传统写法是一个个字段 set，反射可以批量处理。

实战二：自定义注解 + 反射实现字段校验

业务场景：模拟 Hibernate Validator，用注解标记字段校验规则，反射读取注解执行校验。这是反射 + 注解的经典组合，也是框架底层原理的体现。
